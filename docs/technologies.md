# Technologies

List of technologies (frameworks, libraries, and tools) used in this project.

[[toc]]

## Core Technologies

We use the following frameworks for backend development to build scalable and maintainable server-side applications.

### Node.js
A JavaScript runtime built on Chrome's V8 engine, enabling fast and scalable server-side applications.

### NestJS
A progressive Node.js framework for building efficient and modular server-side applications.

## Frameworks and Libraries

### Express
A minimal and flexible Node.js web framework that provides a robust set of features for web and mobile applications.

### TypeScript
A superset of JavaScript that adds static typing for better code reliability and maintainability.

## Database

### Prisma
An ORM for TypeScript/JavaScript that simplifies database access with a type-safe query builder.

### PostgreSQL
A powerful, open-source relational database known for reliability, robustness, and high performance.

## Authentication and Authorization

### JWT (jsonwebtoken)
A library to manage JSON Web Tokens for authentication and authorization.

### Passport & Passport-Google-OAuth2
Libraries that support OAuth2 and various authentication strategies.

## Validation and Parsing

### Zod & NestJS-Zod
Libraries for data validation, object transformation, and serialization using schemas.

## Logging and Monitoring

### Pino
A fast and feature-rich logging library for Node.js with minimal overhead.

## Testing

### Jest
A comprehensive testing framework for unit, integration, and end-to-end tests.

### Supertest
A library for testing HTTP servers by sending requests and asserting responses.

## Package Management

### NPM
A fast and disk-efficient package manager that improves dependency management.

## Linting and Formatting

### ESLint & Prettier
Tools to enforce code style, consistency, and detect potential issues.

## Git Hooks and Commit Management

### Husky, Commitlint, Commitizen, Lint-staged
Tools for managing Git hooks, enforcing conventional commit messages, and linting staged files.

## DevOps and CI/CD

### GitHub Actions
Automates CI/CD workflows: build, test, and deploy directly from GitHub.

### Docker
Runs applications in containers, providing a consistent deployment environment.

### Renovate
Automates dependency updates to keep the project up-to-date.

## Documentation

### Markdown & VuePress
Used to create documentation, README files, and documentation websites.

## Miscellaneous

### dotenv / env-cmd
Manage environment variables and simplify configuration management.
